# Legacy Infrastructure — Windows Server 2016 VM

This Terraform configuration provisions a Windows Server 2016 Datacenter VM on Azure,
representing the "before" state of the PropertyManager application's production environment.

## What It Creates

| Resource | Purpose |
|----------|---------|
| Resource Group (auto-generated) | Container for all legacy resources |
| Virtual Network + Subnet | Isolated network with auto-generated CIDR blocks |
| Public IP (Static) | Static IP for HTTP/RDP access |
| NSG | Firewall rules: HTTP (80), HTTPS (443), RDP (3389 from your current IP only) |
| Windows Server 2016 Datacenter VM (Standard_B2ms) | IIS + SQL Server Express host |
| Data Disk (64 GB) | Dedicated SQL Server data file storage |
| Custom Script Extension | Automated IIS + SQL Server setup |

## Prerequisites

- [Terraform >= 1.5](https://developer.hashicorp.com/terraform/downloads)
- [Azure CLI](https://learn.microsoft.com/en-us/cli/azure/install-azure-cli) authenticated (`az login`)
- An Azure subscription

## Quick Start (Using Task)

From the repository root:

```bash
# 1. Preview what will be created
task legacy:plan

# 2. Provision the VM
task legacy:up

# 3. Get your credentials and connection info
task legacy:creds

# 4. Connect via RDP or browse the site
task legacy:browse
```

## Manual Terraform Usage

If you prefer to run Terraform directly:

```bash
# 1. Initialize Terraform
terraform init

# 2. Review the plan (default region: canadacentral)
terraform plan -var "location=canadacentral"

# 3. Apply
terraform apply -var "location=canadacentral"

# 4. View outputs
terraform output resource_group_name
terraform output -raw vm_public_ip
terraform output -raw rdp_connection_string
terraform output -raw vm_admin_password
terraform output -raw sql_sa_password
terraform output -raw iis_url
```

## Configuration

### Terraform Variables

| Variable | Type | Default | Notes |
|----------|------|---------|-------|
| `location` | string | — | Azure region (required, pass via CLI: `-var "location=canadacentral"`) |
| `tags` | string | "Property Manager" | Application identifier tag |
| `admin_username` | string | "pmadmin" | VM administrator username |

**No terraform.tfvars file is needed.** All sensitive values (VM admin password, SQL SA password) are randomly generated by Terraform.

### Auto-Generated Resources

Resource naming uses randomized names to avoid conflicts:
- Resource Group: `{pet}-{id}_rg` (e.g., `dolphin-a1b2_rg`)
- VM: `{pet}-{id}-vm`
- VNet: `{pet}-{id}-vnet`
- Subnet: `{pet}-{id}-subnet`
- NIC: `{pet}-{id}-nic`
- NSG: `{pet}-{id}-nsg`
- Public IP: `{pet}-{id}-pip`

**Admin password & SQL SA password** are randomly generated (16 chars, include special characters).

## Post-Provisioning

The Custom Script Extension runs during VM creation and:
1. Initializes and formats the data disk (F: drive)
2. Installs IIS with ASP.NET 4.6 features
3. Downloads and installs SQL Server 2016 Express
4. Creates the `PropertyManagement` database on the data disk
5. Configures the IIS site pointing to `C:\inetpub\wwwroot\PropertyManagement`

## Terraform Outputs

```bash
terraform output resource_group_name      # Resource group name
terraform output vm_public_ip             # Static public IP
terraform output rdp_connection_string    # Ready-to-use RDP command
terraform output vm_admin_username        # Administrator username
terraform output -raw vm_admin_password   # Admin password (sensitive)
terraform output -raw sql_sa_password     # SQL SA password (sensitive)
terraform output iis_url                  # URL to access the website
```

## Destroying Resources

Using Task:

```bash
task legacy:down
```

Or manually:

```bash
terraform destroy
az group delete -n <resource_group_name> --yes --no-wait
```

## Security Notes

- **RDP Access:** Restricted to your current IP (auto-detected at provisioning time)
- **Passwords:** All sensitive values are randomly generated and marked `sensitive` in Terraform outputs — never stored in tfvars
- **Never commit Terraform state files** to version control (they contain sensitive data)
- The VM uses a local admin account — not domain-joined
- Update the NSG rule if you need to RDP from a different IP:
  ```bash
  az network nsg rule update -g <rg> --nsg-name <nsg> -n Allow-RDP \
    --source-address-prefixes "YOUR_NEW_IP/32"
  ```
