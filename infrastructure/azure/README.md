# Modern PaaS Infrastructure — Azure App Service + SQL + Blob

This Terraform configuration provisions the migration target for the PropertyManager application:
a fully managed Azure PaaS environment replacing the legacy IIS/SQL Server VM.

## What It Creates

| Resource | Purpose |
|----------|---------|
| Resource Group (auto-generated) | Container for all modern resources |
| App Service Plan (S1, Windows) | Compute for the web application |
| Windows Web App (.NET 4.6) | Hosts the migrated PropertyManager application |
| Azure SQL Server + Database | Managed SQL replacing SQL Server Express |
| Storage Account + Container | Blob storage for migrated file attachments |
| Application Insights + Log Analytics | Monitoring and diagnostics |
| SQL Firewall Rules | Access control for Azure services and dev IPs |

## Prerequisites

- [Terraform >= 1.5](https://developer.hashicorp.com/terraform/downloads)
- [Azure CLI](https://learn.microsoft.com/en-us/cli/azure/install-azure-cli) authenticated (`az login`)
- An Azure subscription

## Quick Start (Using Task)

From the repository root:

```bash
# 1. Preview what will be created
task azure:plan

# 2. Provision the PaaS resources
task azure:up

# 3. View outputs (connection strings, URLs, etc.)
cd infrastructure/azure/
terraform output
```

## Manual Terraform Usage

If you prefer to run Terraform directly:

```bash
# 1. Initialize Terraform
terraform init

# 2. Review the plan
terraform plan -var "location=canadacentral"

# 3. Apply
terraform apply -var "location=canadacentral"

# 4. View outputs
terraform output resource_group_name
terraform output -raw app_service_url
terraform output -raw sql_connection_string
terraform output -raw storage_connection_string
```

## Configuration

### Terraform Variables

| Variable | Type | Default | Notes |
|----------|------|---------|-------|
| `location` | string | — | Azure region (required, pass via CLI: `-var "location=canadacentral"`) |
| `tags` | string | "Property Manager" | Application identifier tag |

**No terraform.tfvars file is needed.** All sensitive values (SQL admin password, storage keys) are randomly generated by Terraform.

### Auto-Generated Resources

Resource naming uses randomized names to avoid conflicts:
- Resource Group: `{pet}-{id}_rg`
- App Service Plan: `{pet}-{id}-asp`
- Web App: `{pet}-{id}-app`
- SQL Server: `{pet}{id}sql`
- Storage Account: `pm{id}storage`

## Deployment

After provisioning, deploy the application:

```bash
# Using Azure CLI (from publish/ directory)
az webapp deployment source config-zip \
  --resource-group <rg-name> \
  --name <app-name> \
  --src-path ./publish.zip
```

Or use Web Deploy if preferred.

## Terraform Outputs

```bash
terraform output resource_group_name           # Resource group name
terraform output -raw app_service_url          # Application URL
terraform output -raw sql_server_fqdn          # SQL Server FQDN
terraform output -raw sql_connection_string    # Connection string (sensitive)
terraform output -raw storage_connection_string # Storage connection (sensitive)
terraform output -raw app_insights_key         # Application Insights key
```

## Architecture Notes

- **Connection Strings** are injected into the App Service via Application Settings
- **Managed Identity** is configured on the App Service (ready for Key Vault integration)
- **Blob Storage** replaces the legacy `varbinary(MAX)` attachment pattern
- **Application Insights** provides monitoring not available in the legacy environment
- **SQL Firewall** allows Azure services by default; add developer IPs as needed

## Destroying Resources

Using Task:

```bash
task azure:down
```

Or manually:

```bash
terraform destroy
az group delete -n <resource_group_name> --yes --no-wait
```

## Security Notes

- **SQL Admin Password:** Randomly generated and marked `sensitive` in outputs
- **Storage Keys:** Marked `sensitive` — rotate periodically
- **Never commit Terraform state files** to version control (they contain sensitive data)
- Use **Azure Key Vault** for production secret management (set `key_vault_id` in variables)
- Enable **Managed Identity** on the App Service for Azure resource authentication
- **Application Insights** requires HTTPS for best security practices — configure SSL certificate
